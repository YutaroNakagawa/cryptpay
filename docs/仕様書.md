---

## 1. システム概要

### 1.1 目的

ECサイト・オンラインサービス事業者（加盟店）が、顧客からの仮想通貨（暗号資産）決済を受け付け、**法定通貨（日本円・米ドル等）で売上を受け取れる**ようにする決済代行プラットフォーム。

### 1.2 法的位置づけ

本システムは「**暗号資産交換業**」に**該当しない**よう設計されている。

| 項目 | 設計方針 |
|------|---------|
| 収益源 | 暗号資産の売買差益ではなく「**システム利用料（手数料）**」 |
| 手数料徴収 | 事前デポジット制（出金時に前払い残高から充当） |
| 価格変動リスク | NOWPayments が即時換金するため加盟店は負わない |
| 資金移動 | NOWPayments の Custody インフラを経由（自社ウォレット管理なし） |

### 1.3 提供価値

- 加盟店は暗号資産の知識なしに仮想通貨決済を導入できる
- ボラティリティリスクなしで日本円・ドルとして売上を受け取れる
- QRコードによる直感的な決済 UI を標準提供
- NOWPayments API 経由で 50+ 通貨に対応

## 3. データベース設計

### 3.1 テーブル一覧

| テーブル名 | 説明 |
|-----------|------|
| `merchants` | 加盟店情報・残高 |

### 3.2 merchants テーブル

| カラム | 型 | 説明 |
|-------|-----|------|

| `name` | TEXT | 店舗名 |
| `email` | TEXT UNIQUE | メールアドレス |
| `fiat_currency` | VARCHAR(3) | 受取通貨（JPY/USD/EUR） |
| `deposit_balance` | DECIMAL(18,2) | 手数料デポジット残高 |
| `sales_balance` | DECIMAL(18,2) | 売上残高 |
| `created_at` | TIMESTAMPTZ | 登録日時 |
| `updated_at` | TIMESTAMPTZ | 更新日時（自動） |

### 3.3 transactions テーブル

| カラム | 型 | 説明 |
|-------|-----|------|
| `id` | UUID PK | トランザクションID |

| `order_id` | TEXT | 加盟店側注文ID（冪等キー） |
| `nowpayments_payment_id` | TEXT UNIQUE | NOWPayments 決済ID |

| `pay_address` | TEXT | 送金先アドレス |
| `description` | TEXT | 商品説明 |
| `created_at` | TIMESTAMPTZ | 作成日時 |
| `completed_at` | TIMESTAMPTZ | 完了日時 |

```

                               → PARTIALLY_PAID
               → FAILED
               → EXPIRED
```

## 4. API 仕様

### 共通仕様

- **ベースURL**: `https://your-domain.com`
- **Content-Type**: `application/json`
- **文字コード**: UTF-8
- **エラー形式**:
  ```json
  { "error": "エラーメッセージ", "code": "ERROR_CODE" }
  ```

---

### 4.1 加盟店登録

**POST `/api/merchants/register`**

認証不要の公開エンドポイント。

リクエスト:
```json
{
  "name": "山田商店",
  "email": "yamada@example.com",
  "fiatCurrency": "JPY"
}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `name` | string | ✓ | 店舗名 |
| `email` | string | ✓ | メールアドレス（一意） |
| `fiatCurrency` | string | — | 受取通貨（デフォルト: JPY） |

レスポンス `201`:
```json
{
  "merchantId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "name": "山田商店",
  "email": "yamada@example.com",
  "fiatCurrency": "JPY"
}
```

エラー `409`: メール重複

---

### 4.2 決済インボイス作成

**POST `/api/payments/create`**

リクエスト:
```json
{
  "merchantId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "amountFiat": 10000,
  "payCurrency": "btc",
  "orderId": "ORDER-001",
  "description": "商品名（任意）"
}
```

| フィールド | 型 | 必須 | 説明 |
|-----------|-----|------|------|
| `merchantId` | UUID | ✓ | 加盟店ID |
| `amountFiat` | number | ✓ | 商品代金（法定通貨） |
| `payCurrency` | string | — | 支払通貨（デフォルト: btc） |
| `orderId` | string | ✓ | 注文ID（重複不可） |
| `description` | string | — | 商品説明 |

レスポンス `200`:
```json
{
  "transactionId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "paymentId": 1234567890,
  "payAddress": "bc1qxy2kgdygjrsqtzq2n0yrf2493p83kkfjhx0wlh",
  "payAmount": 0.00066667,
  "payCurrency": "BTC",
  "priceAmount": 10350,
  "priceCurrency": "JPY",
  "productAmount": 10000,
  "platformFee": 350,
  "status": "PENDING"
}
```

> **手数料計算**: `platformFee = ceil(amountFiat × 3.5%)`  
> `priceAmount = amountFiat + platformFee`（購入者請求額）

---

### 4.3 決済ステータス照会

**GET `/api/payments/{transactionId}`**

- `PENDING/WAITING/CONFIRMING` 状態の場合は NOWPayments API に直接照会してステータスを同期する
- 5秒ごとのポーリングで決済完了を検知できる

レスポンス `200`:
```json
{
  "id": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "status": "FINISHED",
  "amount_fiat": 10000,
  "platform_fee_fiat": 350,
  "pay_currency": "BTC",
  "pay_amount": 0.00066667,
  "pay_address": "bc1q...",
  "completed_at": "2026-04-15T10:30:00.000Z"
}
```

---

### 4.4 デポジット入金

**POST `/api/deposit`**

出金手数料の前払い。出金1件につき ¥500（固定）が必要。

リクエスト:
```json
{
  "merchantId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "amount": 5000,
  "description": "初期デポジット（任意）"
}
```

レスポンス `200`:
```json
{
  "message": "デポジットが入金されました",
  "depositBalance": 5000
}
```

---

### 4.5 出金リクエスト

**POST `/api/payout`**

売上残高を法定通貨で出金する。デポジット残高が手数料（¥500）を下回る場合は `403` を返す。

リクエスト:
```json
{
  "merchantId": "xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
  "amountFiat": 50000,
  "destinationAddress": "出金先アドレス（NOWPayments Payout 仕様に従う）"
}
```

エラー `403` (デポジット不足):
```json
{
  "error": "手数料デポジット残高が不足しています。必要額: 500, 残高: 0。",
  "code": "DEPOSIT_INSUFFICIENT_FOR_WITHDRAWAL_FEE"
}
```

---

### 4.6 加盟店情報取得

**GET `/api/merchant/{merchantId}`**

レスポンス `200`:
```json
{
  "id": "...",
  "name": "山田商店",
  "fiat_currency": "JPY",
  "deposit_balance": 4500,
  "sales_balance": 32000
}
```

---

### 4.7 決済履歴取得

**GET `/api/merchant/{merchantId}/transactions`**

クエリパラメータ: `?limit=50`（デフォルト）

---

### 4.8 Webhook（NOWPayments → クリプトペイ）

**POST `/api/webhooks/payment`**

NOWPayments が決済状態変化時に自動呼び出す。加盟店側での実装は不要。

- HMAC-SHA512 署名を `x-nowpayments-sig` ヘッダーで検証
- 冪等処理（重複受信を `webhook_events` テーブルで防止）
- `finished` 受信時に `transactions.status` を `FINISHED` に更新し `sales_balance` を加算

## 5. セキュリティ設計

| 項目 | 対策 |
|------|------|
| Webhook 認証 | HMAC-SHA512 署名検証（`NOWPAYMENTS_IPN_SECRET`） |
| 管理者 API | `x-admin-key` ヘッダー認証（`ADMIN_API_KEY`） |
| DB アクセス | Supabase Service Role Key（サーバーサイドのみ） |
| 環境変数 | `.env` は Git 管理外（`.gitignore` 設定済み） |
| 入力検証 | Zod スキーマによる全エンドポイントのバリデーション |
| 冪等性 | `nowpayments_payment_id` UNIQUE 制約、`webhook_events` 重複チェック |

## 6. 手数料ポリシー

| 手数料種別 | 金額 | 説明 |
|-----------|------|------|
| プラットフォーム手数料 | 商品代金の **3.5%**（切り上げ） | 購入者が負担（請求額に上乗せ） |
| 出金手数料 | **¥500 固定**（1件あたり） | 事前デポジットから充当 |

> 手数料率・出金手数料は環境変数 `PLATFORM_FEE_RATE` / `WITHDRAWAL_FEE_FIXED` で変更可能。

## 7. 環境変数一覧

| 変数名 | 用途 | 必須 |
|--------|------|------|
| `NEXT_PUBLIC_SUPABASE_URL` | Supabase プロジェクト URL | ✓ |
| `NEXT_PUBLIC_SUPABASE_ANON_KEY` | Supabase Anon Key | ✓ |
| `SUPABASE_SERVICE_ROLE_KEY` | Supabase Service Role Key（サーバーのみ） | ✓ |
| `NOWPAYMENTS_API_KEY` | NOWPayments API キー | ✓ |
| `NOWPAYMENTS_IPN_SECRET` | Webhook 署名検証用 Secret | ✓ |
| `NEXT_PUBLIC_APP_URL` | デプロイ先 URL（Webhook URL に使用） | ✓ |
| `ADMIN_API_KEY` | 管理者 API 認証キー | 推奨 |
| `PLATFORM_FEE_RATE` | 手数料率（%）デフォルト: 3.5 | — |
| `WITHDRAWAL_FEE_FIXED` | 出金手数料（固定額）デフォルト: 500 | — |
| `DEMO_MODE` | サーバーサイドデモモード（NOWPayments モック） | — |
| `NEXT_PUBLIC_DEMO_MODE` | フロントエンドデモモード（デモ完了ボタン表示） | — |

## 8. 非機能要件

| 項目 | 仕様 |
|------|------|
| 対応ブラウザ | Chrome 最新版、Safari 最新版、Firefox 最新版 |
| モバイル対応 | iOS Safari / Android Chrome（レスポンシブ対応） |
| 決済タイムアウト | 30分（NOWPayments 標準） |
| ステータス同期 | 5秒ごとポーリング（Webhook 非到達時の自動フォールバック） |
| データ保持 | Supabase の標準バックアップポリシーに準拠 |
| スケーラビリティ | Vercel / Cloudflare Pages 等のサーバーレス環境で水平スケール可能 |

## 9. 依存サービス

| サービス | 用途 | 障害時の影響 |
|---------|------|------------|
| Supabase | データ永続化 | 決済作成・照会不可 |
| NOWPayments | 決済処理・換金 | 決済作成不可（デモモードで継続可） |
| Vercel 等 | アプリホスティング | サービス全停止 |
