# クリプトペイ 取扱説明書（運営者向け）

このドキュメントは**サービスを運営するあなた向け**です。  
加盟店（納品先の EC サイト運営者）に渡すのは `加盟店向けガイド.md` です。

---

## 登場人物の整理

```
あなた（運営者）
 ├─ Supabase アカウント    ← 全データはここに入る
 ├─ NOWPayments アカウント  ← 仮想通貨の受け取り・換金はここ
 └─ クリプトペイ（このシステム）を運用

加盟店（納品先）
 └─ /setup にアクセスして Merchant ID を取得するだけ
    Supabase も NOWPayments も不要
```

---

## 初期セットアップ（あなたが一度だけやること）

### 1. Supabase でデータベースを作る

1. https://supabase.com でプロジェクトを作成
2. **SQL Editor** を開いて `supabase/migrations/001_initial.sql` の内容を実行
3. **Settings → API** から以下をメモする:
   - Project URL
   - anon public キー
   - service_role キー

### 2. NOWPayments を設定する

1. https://nowpayments.io でアカウント作成・ログイン
2. **API Keys** で API キーを生成してメモ
3. **IPN Settings** に以下を登録:
   ```
   https://あなたのドメイン/api/webhooks/payment
   ```
4. IPN Secret をメモ

### 3. 環境変数を設定してデプロイ

`.env` ファイルに以下を記入:

```env
NEXT_PUBLIC_SUPABASE_URL=（Supabase Project URL）
NEXT_PUBLIC_SUPABASE_ANON_KEY=（Supabase anon public キー）
SUPABASE_SERVICE_ROLE_KEY=（Supabase service_role キー）
NOWPAYMENTS_API_KEY=（NOWPayments API キー）
NOWPAYMENTS_IPN_SECRET=（NOWPayments IPN Secret）
NEXT_PUBLIC_APP_URL=https://あなたのドメイン
ADMIN_API_KEY=（openssl rand -hex 32 で生成）
DEMO_MODE=false
NEXT_PUBLIC_DEMO_MODE=false
```

**Vercel にデプロイする場合:**
```bash
bash scripts/deploy-vercel.sh
```

**Docker で動かす場合:**
```bash
bash scripts/deploy-docker.sh
```

## 加盟店の管理

### 加盟店が自己登録する場合

加盟店に以下の URL を伝えるだけ:

```
https://あなたのドメイン/setup
```

### 管理者として加盟店を手動登録する場合

```bash
curl -X POST https://あなたのドメイン/api/merchants \
  -H "Content-Type: application/json" \
  -H "x-admin-key: 【ADMIN_API_KEY】" \
  -d '{
    "name": "山田商店",
    "email": "yamada@example.com",
    "fiatCurrency": "JPY"
  }'
```

### Supabase で加盟店一覧を確認する

Supabase ダッシュボード → **Table Editor** → `merchants` テーブル

## 手数料の変更

| 項目 | 環境変数 | デフォルト |
|------|---------|----------|
| 決済手数料率 | `PLATFORM_FEE_RATE` | 3.5（%） |
| 出金手数料 | `WITHDRAWAL_FEE_FIXED` | 500（円） |

`.env` を変更して再デプロイすれば反映されます。

## ローカル開発・テスト

```bash
# 環境変数をデモモードに変更
DEMO_MODE=true
NEXT_PUBLIC_DEMO_MODE=true

# 開発サーバー起動
npm run dev
```

デモモードでは NOWPayments への実際のリクエストが行われず、  
決済ページに「（Demo）支払いを完了する」ボタンが表示されます。

## システムの監視・確認

### 決済が届いているか確認

Supabase → Table Editor → `transactions` テーブル

### Webhook が正常に処理されているか確認

Supabase → Table Editor → `webhook_events` テーブル

### NOWPayments 側の状況確認

NOWPayments ダッシュボード → **Transactions**

---

## トラブルシューティング

**決済のステータスが更新されない**  
→ Webhook が届いていない可能性があります。NOWPayments の IPN Settings のURLを確認してください。  
→ クリプトペイはポーリングによる自動フォールバックがあるため、決済ページを開いたまま待てばステータスは更新されます。

**加盟店が「加盟店が見つかりません」エラーになる**  
→ Supabase の `merchants` テーブルにレコードが存在するか確認してください。  
→ Merchant ID が正しいか確認してください。

**出金が「デポジット不足」で失敗する**  
→ `/api/deposit` で加盟店のデポジットを追加してください（出金1件 ¥500）。
